Оновлення та обслуговування проєкту (Update & Rollback Guide)

Цей документ містить чіткі покрокові рекомендації для DevOps та Release інженерів щодо процедури оновлення Telegram-бота the-mix-bot у виробничому (production) середовищі.
1. Підготовка до оновлення
1.1. Створення резервних копій (Бек-ап)

Оскільки проєкт використовує базу даних SQLite з увімкненим режимом Write-Ahead Logging (WAL) для паралельного доступу, просте копіювання файлу бази даних під час роботи бота може призвести до пошкодження архіву. Для створення консистентного знімка (snapshot) необхідно використовувати вбудовану утиліту резервного копіювання.

Виконайте команду для створення резервної копії:
Створення директорії для бекапів, якщо вона відсутня

mkdir -p /opt/the-mix-bot/backups
Безпечне копіювання бази даних "на льоту"

sqlite3 /opt/the-mix-bot/themix.db ".backup '/opt/the-mix-bot/backups/themix_backup_$(date +%Y%m%d_%H%M%S).db'"

Також обов'язково зробіть резервну копію конфігураційного файлу:
```
cp /opt/the-mix-bot/.env /opt/the-mix-bot/backups/env_backup_$(date +%Y%m%d).env

1.2. Планування часу простою та перевірка сумісності

    Ознайомтеся з останніми змінами в коді (комітами). Перевірте, чи не додалися нові змінні у файл .env.example, які потрібно буде перенести у ваш робочий .env.

    Бот використовує метод Long Polling. Під час короткочасного перезапуску служби (до кількох хвилин) повідомлення від користувачів не втрачаються, а накопичуються на серверах Telegram. Вони будуть оброблені відразу після старту оновленої версії бота.

2. Процес оновлення

Крок 1. Зупинка служби бота
Щоб уникнути конфліктів при записі у файли, зупиніть системний сервіс бота:


sudo systemctl stop the-mix-bot

Крок 2. Розгортання нового коду
Перейдіть до директорії проєкту та завантажте останні зміни з гілки main:


cd /opt/the-mix-bot
sudo -u telegram-bot git pull origin main

Крок 3. Оновлення залежностей
Якщо у файлі requirements.txt з'явилися нові бібліотеки, їх потрібно встановити у віртуальне середовище:


sudo -u telegram-bot  -c "source venv/bin/activate && pip install -r requirements.txt"

Крок 4. Оновлення конфігурацій та бази даних

    За необхідності оновіть файл .env.

    Міграція даних: Структура бази даних у цьому проєкті створюється автоматично функцією init_db() при запуску main.py. Якщо ви додали нові таблиці в SQL-скрипти, вони будуть застосовані під час наступного запуску.

Крок 5. Запуск та перевірка після оновлення
Запустіть оновленого бота:


sudo systemctl start the-mix-bot

Одразу перевірте системний журнал на наявність помилок старту (Tracebacks):


sudo journalctl -u the-mix-bot -f -n 50

Надішліть боту команду /start у Telegram для перевірки його працездатності.
3. Процедура відкату (Rollback) у разі невдалого оновлення

Якщо після оновлення бот не запускається, постійно перезавантажується (CrashLoop) або некоректно обробляє дані, негайно виконайте відкат до попередньої робочої версії.

Крок 1. Негайна зупинка служби


sudo systemctl stop the-mix-bot

Крок 2. Відновлення коду
Поверніться до попереднього стабільного коміту в Git:


cd /opt/the-mix-bot
sudo -u telegram-bot git reset --hard HEAD~1

Після цього синхронізуйте залежності зі старим requirements.txt:


sudo -u telegram-bot  -c "source venv/bin/activate && pip install -r requirements.txt"

Крок 3. Відновлення бази даних (Критично важливо)
Якщо нове оновлення пошкодило базу даних, її потрібно відновити з бекапу. При відновленні бази даних у режимі WAL недостатньо просто замінити основний файл .db. Обов'язково видаліть файли журналу (-wal та -shm), інакше база знову пошкодиться.


# Видалення пошкодженої бази та тимчасових файлів журналу транзакцій
sudo -u telegram-bot rm -f /opt/the-mix-bot/themix.db
sudo -u telegram-bot rm -f /opt/the-mix-bot/themix.db-wal
sudo -u telegram-bot rm -f /opt/the-mix-bot/themix.db-shm

# Відновлення чистої копії з бекапу
sudo -u telegram-bot cp /opt/the-mix-bot/backups/themix_backup_YYYYMMDD_HHMMSS.db /opt/the-mix-bot/themix.db

Крок 4. Повторний запуск
Запустіть бота на старій, стабільній конфігурації:


sudo systemctl start the-mix-bot
sudo systemctl status the-mix-bot
