Статичний аналіз та стандартизація коду (Linting)
1. Обраний інструмент та обґрунтування
Для забезпечення високої якості коду, безпеки та стилістичної єдності у проєкті the-mix-bot використовується Ruff — сучасний, написаний на Rust інструмент, що поєднує можливості лінтера та форматера.
Причини вибору:

Виняткова продуктивність: Ruff аналізує всю кодову базу за мілісекунди.

Універсальність: Дозволяє відмовитися від окремого використання Flake8, isort та Black.

Безпека: Вбудована підтримка плагінів flake8-bandit та flake8-async ідеально підходить для захисту асинхронного Telegram-бота.

2. Базові правила та їх значення
Конфігурація інструменту знаходиться у файлі pyproject.toml. До основних правил належать:

E, W (pycodestyle): Базові перевірки відповідності стандарту PEP 8 (пробіли, відступи).

F (Pyflakes): Запобігання використанню невизначених змінних та видалення "мертвого" коду (невикористаних імпортів).

I (isort): Алфавітне та логічне сортування імпортів (стандартні бібліотеки -> сторонні пакети -> локальні модулі).

ASYNC: Аналіз асинхронного коду (виявлення блокуючих викликів всередині async def функцій).

S: Перевірка безпеки (відсутність хардкоду токенів, перевірка SQL-запитів).

3. Інструкція із використання
Усі команди виконуються з кореневої директорії проєкту при активованому віртуальному середовищі.
   1. Запуск повної перевірки (Linting): `ruff check`
   2. Автоматичне виправлення безпечних помилок (Auto-fix): `ruff check --fix`
   3. Автоматичне форматування коду (Formatting): `ruff format`
